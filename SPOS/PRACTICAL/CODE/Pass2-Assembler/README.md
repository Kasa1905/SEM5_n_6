# Pass 2 Assembler

This folder contains the implementation of a Two-Pass Assembler's second pass, which generates machine code from intermediate representation.

## 📋 Program Overview

### **pass2.java** - Pass 2 Assembler Implementation
- **Purpose**: Second pass of two-pass assembler
- **Function**: Machine code generation from intermediate code
- **Input**: Intermediate code (`Output.txt`), Symbol table, Literal table
- **Output**: Final machine code, error reports

## 🎯 Pass 2 Concept

### Two-Pass Assembler - Second Pass
The second pass takes intermediate code generated by Pass 1 and produces final machine code by:
- Replacing symbolic references with actual addresses
- Generating binary/hexadecimal machine instructions
- Resolving all forward references
- Producing relocatable object code

### Pass 2 Responsibilities:
- Process intermediate code tokens
- Replace symbol references with addresses
- Replace literal references with addresses  
- Generate final machine code
- Handle address resolution
- Error reporting for unresolved references

## 📁 Files Structure

### Input Files:
- **`Output.txt`** - Intermediate code from Pass 1
- **Symbol Table** - From Pass 1 (in memory/file)
- **Literal Table** - From Pass 1 (in memory/file)

### Output Files:
- **Machine code** - Final executable code
- **Error report** - Unresolved references, if any

### Intermediate Code Format:

## 📝 Intermediate Code Processing

### Token Types:
- **IS (Imperative Statement)**: Machine instructions
- **DL (Declarative)**: Data declarations
- **AD (Assembler Directive)**: Assembly directives
- **RG (Register)**: Register operands
- **S (Symbol)**: Symbol table references
- **L (Literal)**: Literal table references
- **C (Constant)**: Direct constants

### Sample Processing:
```java
// Input: (IS,04) (RG,1) (S,01)
// Output: 04 1 104 (if symbol A is at address 104)

if (tokenType.equals("IS")) {
    machineCode = instruction_table[code];
    if (hasOperand1) {
        if (operand1Type.equals("RG")) {
            machineCode += " " + operand1Value;
        }
    }
    if (hasOperand2) {
        if (operand2Type.equals("S")) {
            int address = symbol_table[operand2Value].address;
            machineCode += " " + address;
        }
    }
}
```

## 🔄 Algorithm Steps

### Pass 2 Processing:
1. **Initialize**: Load symbol table, literal table, instruction table
2. **Read Intermediate Code**: Parse each line of intermediate code
3. **Token Analysis**: Extract opcode and operand tokens
4. **Address Resolution**:
   - Replace (S,n) with symbol_table[n].address
   - Replace (L,n) with literal_table[n].address
   - Keep (RG,n) and (C,n) as-is
5. **Machine Code Generation**: Convert to binary/hex format
6. **Output**: Write machine code with addresses
7. **Error Check**: Report unresolved references

### Symbol Resolution:
```java
private String resolveSymbol(int symbolIndex) {
    if (symbolIndex < symbol_table.length && 
        symbol_table[symbolIndex] != null) {
        return String.valueOf(symbol_table[symbolIndex].address);
    } else {
        System.err.println("Error: Undefined symbol at index " + symbolIndex);
        return "???";
    }
}
```

### Literal Resolution:

## 📊 Machine Code Format

### Instruction Format:
```
[Opcode] [Reg1] [Operand2]
   8-bit  4-bit   12-bit (address/immediate)
```

### Example Conversions:
| Intermediate Code | Machine Code | Description |
|-------------------|--------------|-------------|
| (IS,04) (RG,1) (S,01) | 04 1 104 | MOVER R1, A |
| (IS,01) (RG,1) (L,01) | 01 1 106 | ADD R1, ='5' |
| (DL,01) (C,1) | - | A DS 1 (space reservation) |
| (AD,02) | - | END (no code generated) |

## 📈 Complexity Analysis

- **Time Complexity**: O(n) where n = intermediate code lines
- **Space Complexity**: O(s + l) where s = symbols, l = literals
- **Symbol Lookup**: O(1) with direct indexing
- **File Processing**: O(n) for reading intermediate code

## 🛠️ Instruction Set

### Machine Instructions:
| Mnemonic | Opcode | Format | Description |
|----------|--------|---------|-------------|
| STOP | 00 | (00) | Halt execution |
| ADD | 01 | (01) reg, mem | Add memory to register |
| SUB | 02 | (02) reg, mem | Subtract memory from register |
| MUL | 03 | (03) reg, mem | Multiply register by memory |
| MOVER | 04 | (04) reg, mem | Move memory to register |
| MOVEM | 05 | (05) reg, mem | Move register to memory |
| COMP | 06 | (06) reg, mem | Compare register with memory |
| BC | 07 | (07) cond, addr | Branch on condition |
| DIV | 08 | (08) reg, mem | Divide register by memory |
| READ | 09 | (09) mem | Read into memory |
| PRINT | 10 | (10) mem | Print from memory |

## 🔍 Error Handling

### Common Errors:
1. **Undefined Symbols**: Symbol reference not in symbol table
2. **Undefined Literals**: Literal reference not in literal table
3. **Invalid Tokens**: Malformed intermediate code
4. **Array Bounds**: Index out of range

### Error Recovery:
```java
try {
    if (symbolIndex >= 0 && symbolIndex < symbol_table.length 
        && symbol_table[symbolIndex] != null) {
        address = symbol_table[symbolIndex].address;
    } else {
        System.err.println("Warning: Undefined symbol at index " + symbolIndex);
        address = 0; // Default value
    }
} catch (Exception e) {
    System.err.println("Error processing symbol: " + e.getMessage());
}
```
